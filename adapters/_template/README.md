# Шаблон адаптера

Копируйте этот каталог, когда создаёте новый адаптер.

Шаблон намеренно маленький, чтобы будущие адаптеры оставались лёгкими.

## Ожидаемый рабочий процесс

1. Скопируйте `_template/` в `adapters/<название-платформы>/`.
2. Заполните `adapter.yaml`.
3. Добавьте основной файл инструкций платформы.
4. Добавьте локальные справочные файлы только если они реально улучшают работу платформы.
5. Проверьте адаптер на `../../core/eval-cases.md`, прежде чем переводить его в `active`.

## Каким должен быть первый вариант

Первая версия адаптера обычно должна включать:

- `adapter.yaml`;
- один основной файл инструкций;
- от нуля до двух справочных файлов;
- от нуля до нескольких файлов платформенных метаданных.

Если для первого варианта нужно заметно больше, проверьте, не относится ли часть этой сложности на самом деле к `core/`.
