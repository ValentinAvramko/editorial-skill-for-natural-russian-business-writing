# Шаблон адаптера

Копируйте этот каталог, когда создаёте новый слой доставки поверх `core/`.

Шаблон намеренно маленький, чтобы будущие адаптеры оставались лёгкими и не превращались в параллельный источник правды.

## Ожидаемый рабочий процесс

1. Скопируйте `_template/` в `adapters/<имя-адаптера>/`.
2. Заполните `adapter.yaml`.
3. Добавьте один основной entrypoint.
4. Добавьте локальные `references/` только если они реально нужны во время работы.
5. Добавьте install, usage или config-файлы только если этого требует способ доставки.
6. Проверьте адаптер на `../../core/eval-cases.md`, прежде чем переводить его в `active`.

## Каким должен быть первый вариант

Первая версия адаптера обычно должна включать:

- `adapter.yaml`;
- один основной файл инструкций;
- от нуля до двух справочных файлов;
- минимум supporting docs или config.

Если для первого варианта нужно заметно больше, проверьте, не относится ли часть этой сложности на самом деле к `core/`.
