# Использование prompt в Gemini

`PROMPT.md` рассчитан на ручную вставку в обычный чат без дополнительных файлов.
`references/` используются как дополнительные текстовые материалы: Gemini не откроет их по имени файла, поэтому нужный файл или его релевантный фрагмент надо вставлять в диалог вручную.

## Базовый сценарий

1. Открой новый диалог.
2. Скопируй `PROMPT.md` целиком.
3. Сразу под ним добавь задачу и исходный текст.

Пример:

```text
Это короткое рабочее сообщение.
Нужен только итоговый вариант.

Текст:
...
```

## Если модель начинает объяснять вместо редактирования

Повтори ограничение отдельным сообщением:

```text
Верни только итоговый отредактированный текст без пояснений и без комментариев о проделанной работе.
```

## Как использовать references

Подключай `references` после основного `PROMPT.md`, когда нужно точнее направить модель.

- Используй [`adapters/prompt/references/patterns.md`](../references/patterns.md), если текст слишком гладкий, канцелярский, карьерно-шаблонный или явно нейросетевой.
- Используй [`adapters/prompt/references/examples.md`](../references/examples.md), если важно удержать степень вмешательства и попасть в правильную глубину редактуры.
- Для большинства случаев достаточно вставить только релевантный раздел, а не весь файл.
- Целиком reference-файл имеет смысл вставлять только для сложных и пограничных случаев.

Практический порядок:

1. Вставь `PROMPT.md`.
2. Коротко сформулируй задачу.
3. Вставь нужный reference-фрагмент или reference-файл.
4. Ниже добавь исходный текст.

Пример с `patterns.md`:

```text
Ниже даю список формулировок и паттернов, на которые здесь особенно стоит смотреть.
Используй его как ориентир, но переписывай текст своими словами.

[вставка нужного фрагмента из patterns.md]

Текст:
...
```

Пример с `examples.md`:

```text
Ниже даю before/after-примеры как ориентир по степени редактуры.
Смотри на глубину вмешательства и тон, но не копируй примеры буквально.

[вставка нужного фрагмента из examples.md]

Текст:
...
```

## Если нужна минимальная правка

Добавь перед текстом:

```text
Если исходник уже нормальный, правь минимально.
Не делай текст более гладким, чем нужно.
```

## Что считать хорошим результатом

- тон остался деловым и человеческим;
- текст не ушел в разговорность или канцелярит;
- формулировки стали точнее без выдуманных деталей;
- ответ не содержит мета-комментариев модели.
