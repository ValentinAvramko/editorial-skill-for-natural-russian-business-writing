# Черновик release notes для `v1.4.0`

Ниже — короткий текст, который можно использовать как основу для GitHub Release.

## Заголовок

`v1.4.0 — строже работа с англицизмами, управленческой абстракцией и карьерными шаблонами`

## Короткое описание

`v1.4.0` усиливает редакторский стандарт для русскоязычного делового текста. Релиз лучше распознаёт гибридные русско-английские конструкции, синтетические управленческие формулировки, формулы карьерной релевантности и перегруженные технические списки.

Цель изменений — не запретить профессиональную терминологию, а точнее отличать нужные термины от жаргона и декоративной деловой упаковки.

## Что входит в релиз

- Добавлены правила про англицизмы в русском деловом тексте: английские термины сохраняются, когда они точны и ожидаемы для адресата, но не используются ради мнимой профессиональности.
- Расширен справочник паттернов: добавлены гибридные конструкции вроде `держать delivery`, `убирать blockers`, `брать ownership`.
- Добавлены паттерны синтетических управленческих формулировок, где действие спрятано за абстрактной упаковкой.
- Добавлены карьерные формулы релевантности, которые заменяются на прямое описание похожих задач из исходника.
- Усилен контроль повторов профессиональных опорных слов в соседних предложениях и абзацах.
- Добавлены правила для плотных технических списков: стек не удаляется без основания, а группируется или связывается с задачей, архитектурой, процессом или результатом.
- Добавлены новые примеры 19-22, которые калибруют эти сценарии.

## Проверка перед выпуском

- `python build/build_adapters.py check` проходит успешно;
- `core/` остаётся источником правды для новых редакторских правил;
- `adapters/skill/` и `adapters/prompt/` синхронизированы с `core/`;
- `README.md` и служебные build-документы не требуют изменения, потому что упаковка и пользовательский сценарий установки не менялись.
