# Черновик release notes для `v1.3.0`

Ниже — короткий текст, который можно использовать как основу для GitHub Release.

## Заголовок

`v1.3.0 — prompt-пакет разделён на quick chat и instructions-слой`

## Короткое описание

`v1.3.0` не меняет сам редакторский стандарт в `core/`, а делает prompt-адаптер заметно удобнее для реального использования в разных средах.

Вместо одного общего prompt-файла релиз вводит разделённый prompt-пакет: самодостаточный quick-chat entrypoint для обычных диалогов, отдельный тонкий `instructions`-слой для ChatGPT Projects, GPTs и Gemini Gems, а также плоские reference-файлы с предсказуемыми именами.

## Что входит в релиз

- `adapters/prompt/PROMPT.md` заменён на `adapters/prompt/humanize-russian-business-text.prompt.md` как основной самодостаточный prompt для обычных чатов;
- добавлен `adapters/prompt/humanize-russian-business-text.instructions.md`, который не дублирует спецификацию, а требует обязательного использования основного prompt-файла в project- и gem-сценариях;
- prompt reference-файлы вынесены в плоский слой `adapters/prompt/humanize-russian-business-text.patterns.md` и `adapters/prompt/humanize-russian-business-text.examples.md`;
- `build/build_adapters.py` теперь пересобирает и проверяет разделённый prompt-пакет, а не только единый prompt-файл;
- обновлены `build/manifests/prompt.yaml`, `build/README.md`, `README.md` и `test/smoke-checklist.md`, чтобы новая схема установки и проверки была согласована по всему репозиторию.

## Что это меняет на практике

- обычный quick-chat сценарий остаётся простым: можно вставить один prompt-файл и сразу работать;
- для ChatGPT Projects, GPTs и Gemini Gems появляется корректный слой `instructions`, который не размывает редакторскую спецификацию и не заставляет дублировать большой prompt в настройках;
- reference-файлы стало проще подключать вручную и использовать в file-backed сценариях;
- `skill` и `prompt` остаются согласованными по редакторской политике, но prompt-упаковка стала лучше соответствовать реальным форматам использования.

## Проверка перед выпуском

- `python build/build_adapters.py check` проходит успешно;
- `build/manifests/skill.yaml` и `build/manifests/prompt.yaml` соответствуют текущей структуре;
- `README.md` и `build/README.md` описывают актуальную схему prompt-пакета;
- `adapters/skill/agents/openai.yaml` остаётся на актуальном runtime-пути `agents/openai.yaml`.

```md
- один общий `PROMPT.md` заменён на самодостаточный quick-chat prompt-файл;

```
