# План развития

Этот файл нужен для отдельных рабочих планов по развитию репозитория.

Он не описывает текущую пользовательскую документацию и не дублирует README.

## Недавно сделано

- Добавлен лёгкий автоматизированный сценарий оценки поверх `core/eval-cases.md` и `core/eval-rubric.md`.

4. Сделать короткий smoke-checklist для ручной проверки адаптеров перед выпуском.

6. Оформить Topics и Website для GitHub.
7. Подготовить хороший текст для первого pinned post / LinkedIn-анонса.

## Принципы ведения планов

- Каждый крупный адаптер идёт отдельным пунктом.
- README описывает текущее состояние и способ использования, а не roadmap.
- Если план становится объёмным, его лучше выносить в отдельный файл внутри каталога `планы/`.
