# План развития

## Что уже сделано

- `core/` оформлен как единый источник правды для редакторской логики.
- Базовая редакторская спецификация, паттерны, примеры, eval-кейсы и рубрика разнесены по отдельным файлам.
- `adapters/skill/SKILL.md` и `adapters/prompt/PROMPT.md` синхронизированы с `core/prompt-spec.md`.
- `adapters/prompt/PROMPT.md` доведён до самодостаточного portable prompt без обязательных ссылок на локальные reference-файлы, а сценарии использования сведены в основной `README.md`.
- Для skill-а сохранён единый переносимый формат для Codex и Claude Code.
- Поддерживается локальный слой `references/` для калибровки паттернов и глубины правки.
- Добавлен короткий smoke-checklist для ручной проверки `skill` и `prompt` перед выпуском.
- Добавлен лёгкий eval-runner для повторяемых ручных прогонов по `core/eval-cases.md`.

## Что было решено по архитектуре

- `core/` остаётся единственным местом, где меняется редакторская политика.
- `adapters/` содержит только готовые deliverable-пакеты без служебной документации и внутренних манифестов.
- Служебный слой сборки вынесен в `build/`.
- Всё, что относится к тестированию и проверке, вынесено в `test/`.
- Install-инструкции и пользовательские сценарии использования сведены в `README.md`, чтобы не плодить дублирующиеся документы.
- Решение по drift принято в пользу лёгкого вспомогательного build/check-скрипта: `build/build_adapters.py` пересобирает `SKILL.md`, `PROMPT.md` и локальные `references/` из `core/`.

## Ближайшие шаги

- Проверить, не нужен ли отдельный publish-слой для внешней публикации skill-а и prompt-а.
- При новых редакторских изменениях сначала обновлять `core/`, потом прогонять `python build/build_adapters.py sync`.
- Поддерживать smoke и eval в `test/` как единое место для быстрой проверки перед выпуском.
