# Черновик release notes для `v1.0.0`

Ниже — короткий текст, который можно использовать как основу для GitHub Release.

## Заголовок

`v1.0.0 — первый стабильный выпуск редакторского инструмента для естественного делового письма на русском`

## Короткое описание

`v1.0.0` — первый стабильный выпуск репозитория с редакторским стандартом для переписывания русскоязычного делового текста без нейрояза, канцелярита и корпоративно-шаблонного тона.

Инструмент помогает сделать текст естественнее, яснее и профессиональнее, не подменяя факты, намерение и деловую логику.

## Что входит в релиз

- ядро в `core/` с редакторской спецификацией, паттернами, примерами, кейсами и рубрикой оценки;
- portable `skill` для Codex и Claude Code;
- portable `prompt` для ChatGPT, Gemini и похожих чатов;
- лёгкий `eval_runner.py` для повторяемой ручной оценки;
- `build_adapters.py` для проверки и синхронизации производных адаптеров;
- smoke-checklist и release-checklist для ручной проверки перед выпуском.

## Для кого этот релиз

- для специалистов, которые пишут деловые тексты на русском;
- для кандидатов, которые редактируют отклики, письма и сообщения рекрутерам;
- для команд и prompt-инженеров, которым нужен воспроизводимый редакторский стандарт, а не одноразовый промпт.

## Ключевые принципы

- сохранять смысл, факты и намерение;
- не придумывать достижения, мотивацию, числа и новые детали;
- не переисправлять текст, если он уже звучит нормально;
- убирать не только явные AI-штампы, но и синтетическую гладкость, карьерный шаблон и пустую абстрактность.

## Что важно перед использованием

- `core/` остаётся единственным источником правды для редакторской логики;
- `adapters/` меняют только упаковку под конкретную среду;
- для быстрой проверки перед выпуском используйте `adapters/smoke-checklist.md`;
- для системного сравнения версий используйте `core/eval-cases.md` и `core/eval-rubric.md`.
