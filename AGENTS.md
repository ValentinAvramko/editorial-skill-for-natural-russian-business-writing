# Инструкции для агента

## Язык и документация

- Веди диалог с пользователем на русском языке.
- Пользовательские документы и инструкции по установке или использованию держи на русском языке.
- Нейминг каталогов, файлов и структуры репозитория оставляй на английском.
- Если понадобится англоязычная версия документации, не смешивай её с русскоязычным основным слоем: предлагай отдельный экспортный или публикационный слой.

## Границы документов

- Не смешивай в `README.md` и других пользовательских документах внутренние инструкции для агента, правила сопровождения репозитория и рабочие договорённости.
- Все локальные инструкции для агента храни в `AGENTS.md`.
- `README.md` должен оставаться пользовательским документом: что это за инструмент, когда он нужен и как его использовать.
- `plans/` используй для рабочих планов, а не для пользовательской документации.

## Источник правды

- `core/` — единственный источник правды для редакторской логики.
- Если меняется редакторская политика, сначала обновляй `core/`, затем синхронизируй производные файлы в `adapters/`.
- `adapters/` меняют только упаковку под конкретную среду и не должны менять сам редакторский стандарт.

## Git

- После любых изменений в репозитории всегда предлагай сделать `commit` и `push`.
- Если пользователь пишет в явной форме `делай коммит и пуш`, выполняй `commit` и `push` без дополнительного уточнения.
