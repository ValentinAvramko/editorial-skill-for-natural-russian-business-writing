# Шаблон адаптера

Используйте этот шаблон, когда создаёте новый слой доставки поверх `core/`.

Шаблон намеренно маленький, чтобы будущие адаптеры оставались лёгкими и не превращались в параллельный источник правды.

## Ожидаемый рабочий процесс

1. Скопируйте `build/template/adapter.yaml` в `build/manifests/<имя-адаптера>.yaml`.
2. Создайте готовый пакет в `adapters/<имя-адаптера>/`.
3. Заполните манифест.
4. Добавьте один основной entrypoint или два согласованных entrypoint-файла, если доставка разделяет quick-chat и settings/instructions.
5. Добавьте локальные reference-файлы только если они реально нужны во время работы.
6. Добавьте platform-specific config только если этого требует способ доставки.
7. Проверьте адаптер на `core/eval-cases.md`, прежде чем переводить его в `active`.

## Каким должен быть первый вариант

Первая версия адаптера обычно должна включать:

- один основной файл инструкций или короткую согласованную пару файлов;
- от нуля до двух справочных файлов;
- минимум platform-specific config;
- отдельный манифест в `build/manifests/`.

Если для первого варианта нужно заметно больше, проверьте, не относится ли часть этой сложности на самом деле к `core/`.
